Example Description

This example shows how to configure the TIM1 peripheral to generate two different signals with two different frequencies.
The system clock works at 96Mhz and is generated through PLL configuration. The PLL clock source is an external 8Mhz crystal oscillator. The frequency division ratio of APB1 is 1, and the frequency division ratio of APB2 is 1.
The TIM1CLK frequency is set to SystemCoreClock，and we want to get TIM1 counter clock at 96 MHz,so the Prescaler is computed as following:
-Prescaler=（TIM1CLK / TIM1 counter clock）- 1
The TIM1->CCR1 register value is equal to 32768：
CC1 update rate = TIM1 counter clock / CCR1VAL = 1.46KHz，
so the TIM1 Channel 1 generates a periodic signal with a frequency equal to1.46KHz.
The TIM1->CCR2 register value is equal to 16384：
CC2 update rate = TIM1 counter clock / CCR2VAL = 2.92KHz
so the TIM1 Channel 2 generates a periodic signal with a frequency equal to 2.92KHz.
The TIM1->CCR3 register value is equal to 16384：
CC3 update rate = TIM1 counter clock / CCR3VAL = 5.85KHz
so the TIM1 Channel 3 generates a periodic signal with a frequency equal to 5.85KHz.
The TIM1->CCR4 register value is equal to 16384：
CC4 update rate = TIM1 counter clock/ CCR4VAL = 11.7KHz
so the TIM1 Channel 4 generates a periodic signal with a frequency equal to 11.7KHz.

Hardware and Software environment
  • Hardware environment
    • HSE crystal is 8MHz
    • Connect the TIM1 pins to an oscilloscope to monitor the different waveforms:
      • TIM1_CH1 pin (PA.08)
      • TIM1_CH2 pin (PA.09)
      • TIM1_CH3 pin (PA.10)
      • TIM1_CH4 pin (PA.11)
How to use it ?

In order to make the program work, you must do the following :

  • Open TIM_TIM1_OCToggle.uvprojx from this example folder by Keil MDK5
  • Rebuild all files and load your image into target memory

  • Run the example
